Kurzanleitung für Template literals

Wozu braucht es diese WEB-API

Im Allgemeinen erkennt man Template literals an den Backtick-Zeichen (`). Template literals werden in allen modernen Browsern unterstützt, abgesehen vom Internet Explorer. Des Weiteren können auch einfache Operatoren wie Vergleiche sowie Funktionen verwendet werden. Die gängisten Vorteile sind:

  • Template literals für Strings
  • Multi-Line-Strings
  • String-Interpolation mit eingebetteten Ausdrücken
  • spezielle Konstrukte, sogenannte getaggte Templates
  • HTML einfügen

Template literals für Strings

Um einfache ' ' oder doppelte " " Anführungszeichen innerhalb Zeichenfolge nutzen zu können, verwendet man gewöhnlich das Escape-Zeichen \: 

 
  console.log('A \'quote\' inside a string');
  console.log("Another \"quote\" inside a string");

Template literals vereinfachen diese Schreibweise und tragen zu einer besseren Übersichtlichkeit des Codes bei. Zitate etc. können beliebig eingefügt werden: 


  console.log(`This is a string with a 'quote' in it`);
  console.log(`This is a string with a "double quote" in it`);
