Kurzanleitung für Template literals

Wozu braucht es diese WEB-API

Im Allgemeinen erkennt man Template literals an den Backtick-Zeichen (`). Diese sind rechts vom ß. Template literals werden in allen modernen Browsern unterstützt, abgesehen vom Internet Explorer. Des Weiteren können auch einfache Operatoren wie Vergleiche sowie Funktionen verwendet werden. Die gängisten Vorteile sind:

  1. Template literals für Strings
  2. Multi-Line-Strings
  3. String-Interpolation
  4. Tagged Templates
  5. Raw Strings
  6. Einfügen von HTML-Elementen

1. Template literals für Strings

Um einfache ' ' oder doppelte " " Anführungszeichen innerhalb Zeichenfolge nutzen zu können, verwendet man gewöhnlich das Escape-Zeichen \: 

 
  console.log('A \'quote\' inside a string');
  console.log("Another \"quote\" inside a string");

Template literals vereinfachen diese Schreibweise und tragen zu einer besseren Übersichtlichkeit des Codes bei. Zitate etc. können beliebig eingefügt werden: 


  console.log(`This is a string with a 'quote' in it`);
  console.log(`This is a string with a "double quote" in it`);

2. Multi-Line-Strings

Üblicherweise werden mehrzeilige Zeichenfolgen mit folgender Syntax dargestellt: 


  console.log("string text line 1\n" + "string text line 2");

Template literals vereinfachen die Darstellung mehrerer Zeilen. Alle im Code definierten Zeilenumbrüche werden im Browser entsprechend ausgegeben. 


  console.log(`string text line 1
  string text line 2`);

3. String Interpolation

Template literals vereinfachen die Verkettung von Ausdrücken mit Zeichenfolgen, für welche normalerweise der Additionsoperator + genutzt wird:

 
  const a = 5;
  const b = 10;
  console.log("Fifteen is " + (a + b) + " and not " + (2 * a + b) + ".");

Für eine bessere Übersicht bzw. Lesbarkeit des Codes werden Platzhalter der Form ${...} verwendet, um Variablen und Ausdrücke in die Zeichenfolge einzusetzen. Die Ausdrücke werden als String eingefügt: 


  const a = 5;
  const b = 10;
  console.log(`Fifteen is ${a + b} and not ${2 * a + b}.`);

4. Tagged Templates

Die Verknüpfung einer Funktion mit einem Templateliteral wird als Tagged Template Literal bezeichnet. In diesem Fall wird die Funktion als Tag bezeichnet. Hierfür verwendet JavaScript eine spezielle Syntax, die den Funktionsnamen direkt vor dem Templateliteral schreibt.

Das erste Argument einer Tag-Funktion enthält ein Array von Zeichenfolgenwerten. Die restlichen Argumente beziehen sich auf die Ausdrücke. Die Tag-Funktion kann dann beliebige Operationen mit diesen Argumenten ausführen und den manipulierten String zurückgeben. 

 
function myTag(strings, value) {
    console.log(strings); 
    console.log(value); 
      
  return strings[0] + value + strings[1]; }
  const message = myTag`Hello ${"world"}!`;
      
  console.log(message);

In diesem Beispiel fügt die myTag-Funktion die Strings und Werte zusammen, um die vollständige Nachricht zu bilden, und gibt sie zurück.

5. Raw Strings

Raw strings ermöglichen die Ausgabe von Strings ohne die Verarbeitung von Escape-Sequenzen: 

 
    const string = String.raw `string \n within a line`;
    console.log(string);

6. Einfügen von HTML-Elementen

Mit Template literals können auch sämtliche HTML-Elemente auf einer Webseite ausgegeben werden: 


    console.log(`<ul>
    <li> example </li>
    <li> example </li>
    <li> example </li>
    </ul>`);
